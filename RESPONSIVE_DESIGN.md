# 响应式设计说明 - 登录页

## 当前实现方案

登录页采用了**Flexbox + 动态视口高度**的响应式方案，确保在不同手机型号上保持相似的比例和良好的用户体验。

## 核心策略

### 1. 使用 Flexbox 布局
- `.container` 使用 `display: flex; flex-direction: column;`
- `.bottomCard` 使用 `margin-top: auto` 自动推到底部
- 这样可以在不同屏幕高度下自动调整，保持视觉平衡

### 2. 动态视口高度（dvh）
- 使用 `height: 100dvh` 替代 `100vh`
- `dvh`（Dynamic Viewport Height）会根据浏览器工具栏的显示/隐藏自动调整
- 解决了移动端浏览器地址栏/工具栏影响高度的问题

### 3. 最小高度 + 百分比结合
- 大屏设备（≥700px）：使用 `min-height: 37.2%` 保持 Figma 设计的比例（100% - 62.8%）
- 小屏设备（<700px）：使用 `min-height: 320px` 确保内容有足够空间
- 这样既保证了视觉比例，又确保了内容可读性

## 适配的不同设备

### iPhone SE (568px 高度)
- 使用固定最小高度 300px
- 减小按钮间距到 16px
- 调整顶部内边距到 32px

### iPhone 标准尺寸 (667px - 844px 高度)
- 使用 35% - 37.2% 的百分比高度
- 保持原始设计比例

### iPhone Pro Max (932px 高度)
- 使用 37.2% 百分比高度
- 增加顶部内边距到 48px，视觉更平衡

### 横屏模式
- 取消最小高度限制
- 使用固定 flex 值
- 减小间距和内边距

## 技术细节

### 视口高度单位对比

| 单位 | 说明 | 使用场景 |
|------|------|----------|
| `100vh` | 静态视口高度，不随浏览器工具栏变化 | 桌面端、固定布局 |
| `100dvh` | 动态视口高度，随浏览器工具栏自动调整 | **移动端推荐**（当前使用） |
| `100svh` | 小视口高度，始终包含浏览器工具栏 | 不推荐 |

### Flexbox vs 绝对定位

| 方案 | 优点 | 缺点 | 当前使用 |
|------|------|------|----------|
| 绝对定位 (`top: 62.8%`) | 精确还原 Figma 设计 | 在不同屏幕高度下可能不够灵活 | ❌ |
| Flexbox (`margin-top: auto`) | 自动适应不同屏幕，响应式更好 | 需要配合 min-height 控制比例 | ✅ |

## 测试建议

在不同设备上测试时，关注以下几点：

1. **背景动画是否完整显示**
   - 确保底部卡片不会遮挡过多背景内容
   - 背景区域应该有足够的视觉空间

2. **按钮是否完全可见且可点击**
   - 确保所有按钮都在可视区域内
   - 触摸目标大小符合移动端规范（≥44px）

3. **安全区域适配**
   - iPhone X 及以后的设备有安全区域（刘海、底部指示条）
   - 使用 `env(safe-area-inset-bottom)` 确保内容不被遮挡

4. **横屏模式**
   - 横屏时布局应该仍然可用
   - 内容不应该被过度压缩

## 常见问题

### Q: 为什么使用 `min-height` 而不是固定 `height`？
A: `min-height` 允许卡片在内容较多时自动扩展，同时确保最小空间。固定 `height` 可能导致内容溢出。

### Q: 为什么不直接使用 `top: 62.8%`？
A: 百分比定位在不同屏幕高度下虽然保持比例，但在极端情况下（横屏、小屏）可能导致内容显示问题。Flexbox 方案更灵活、更稳定。

### Q: `100dvh` 和 `100vh` 有什么区别？
A: `100vh` 是固定的视口高度，移动端浏览器工具栏显示/隐藏时不会改变。`100dvh` 会动态调整，更符合移动端使用场景。
