# 图片下载问题说明

## ⚠️ 当前状态

Figma MCP 服务器已运行（端口 3845 被占用），但直接访问图片 URL 时返回 **500 错误**：

```
HTTP/1.1 500 Internal Server Error
Error getting image
```

## 🔍 问题分析

### 可能的原因

1. **图片路径不正确**
   - 图片 ID 可能已过期或不存在
   - Figma 设计文件可能已更新，图片 ID 已改变

2. **需要通过 MCP 协议访问**
   - Figma MCP 服务器可能不支持直接 HTTP 访问 `/assets/` 路径
   - 可能需要通过 Cursor 的 MCP 功能来获取资源

3. **权限或认证问题**
   - 可能需要 Figma API Token
   - 或者需要特定的认证头

## ✅ 解决方案

### 方案 1：从 Figma 设计文件直接导出（推荐）

这是最可靠的方法，不依赖 MCP 服务器：

1. **打开 Figma Desktop 应用**
2. **打开对应的设计文件**
3. **找到对应的图片元素**：
   - `e503f988099e962c10dd595d7fb80340d7487ce9.png` - 聊天背景图
   - `b4eb843683aafc4dd42a29a982a4295167ad755a.png` - 教程插图
   - `708-17842.png` - 预览图
   - `8c1d33aa44d06566d5842aed8fc0fd8a8ea89fd7.png` - 拍摄图

4. **导出图片**：
   - 选择图片元素
   - 右侧面板 → "Export"（导出）
   - 选择 PNG 格式
   - 点击 "Export" 按钮

5. **保存到项目目录**：
   ```bash
   # 将导出的图片移动到 public/images/ 目录
   mv ~/Downloads/xxx.png /Users/wangjianle/OCTA/Code/OCTA/octa-web/public/images/
   ```

6. **重命名文件**（如果需要）：
   ```bash
   cd /Users/wangjianle/OCTA/Code/OCTA/octa-web/public/images
   mv 导出的文件名.png e503f988099e962c10dd595d7fb80340d7487ce9.png
   ```

### 方案 2：通过 Cursor MCP 功能获取

如果 Cursor 的 MCP 功能支持获取图片资源：

1. 在 Cursor 中使用 MCP 命令
2. 通过 MCP 协议请求图片资源
3. 保存到 `public/images/` 目录

（具体操作取决于 Cursor MCP 的实现）

### 方案 3：检查图片 ID 是否正确

如果图片 ID 不正确，需要：

1. 在 Figma 中找到对应的图片元素
2. 查看图片的实际 ID 或导出名称
3. 更新代码中的图片路径

## 📝 验证步骤

下载完成后，验证图片是否存在：

```bash
cd /Users/wangjianle/OCTA/Code/OCTA/octa-web/public/images
ls -lh *.png
```

应该看到 4 个 PNG 文件，每个文件大小都不应为 0。

## 🎯 推荐操作

**建议使用方案 1**（从 Figma 直接导出），因为：

1. ✅ 最可靠，不依赖外部服务
2. ✅ 可以确保图片是最新的
3. ✅ 可以控制图片质量和格式
4. ✅ 适合分享给同事（图片包含在项目中）
