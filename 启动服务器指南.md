### 方式 C：检查是否已经有进程在运行

```bash
# 检查 3845 端口是否已被占用

# 或者使用

```

如果端口已被占用，说明服务器已经在运行，可以直接进行步骤 2。

## 步骤 2：验证服务器运行

在浏览器中访问以下 URL，检查服务器是否正常：

```
http://localhost:3845/assets/e503f988099e962c10dd595d7fb80340d7487ce9.png
```

如果看到图片，说明服务器正常运行。

或者使用命令行测试：

```bash
curl -I http://localhost:3845/assets/test 2>&1 | head -1
```

如果返回 HTTP 状态码（如 200 或 404），说明服务器在运行。

## ⚠️ 如果无法启动服务器

如果无法启动 `localhost:3845` 服务器，您可以使用以下替代方案：

### 替代方案 1：从 Figma 设计文件导出

1. 打开 Figma 设计文件
2. 找到对应的图片元素
3. 选择图片 → 导出为 PNG
4. 保存到 `public/images/` 目录，文件名需匹配

### 替代方案 2：从其他来源获取

- 如果您之前下载过这些图片，可以从其他地方复制
- 确保文件名完全匹配（区分大小写）

### 替代方案 3：使用占位符（临时）

可以创建简单的占位符图片（纯色或渐变背景），暂时使用，后续再替换。

## 需要帮助？

如果遇到问题：

1. **服务器无法启动**：检查 Figma MCP 的配置和文档
2. **端口被占用**：使用 `lsof -i :3845` 查看占用进程
3. **连接被拒绝**：检查防火墙设置，确保 3845 端口未被阻止
4. **图片下载失败**：检查服务器日志，确认图片路径是否正确
