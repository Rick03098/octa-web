## 模块定位

常量定义层，集中管理应用中的所有静态常量和配置值。

## 核心逻辑

- **集中管理**: 所有常量统一定义，避免魔法数字/字符串
- **易于维护**: 修改常量只需一处更新
- **类型安全**: 使用 TypeScript `as const` 确保类型推断

## 架构约束

### 严禁
- **严禁在代码中硬编码字符串**: 必须定义为常量
- **严禁可变常量**: 使用 `const` 和 `as const`

### 必须
- **必须使用 UPPER_SNAKE_CASE**: 常量命名规范
- **必须导出**: 供其他模块使用
- **必须分类**: 按功能分组常量

## 成员清单

### **strings.ts** - UI Text Constants
- **依赖**: 无
- **导出**: 所有 UI 文本常量
- **职责**: 定义界面显示的文本字符串
- **分类**:
  - 页面标题
  - 按钮文本
  - 提示消息
  - 错误消息
